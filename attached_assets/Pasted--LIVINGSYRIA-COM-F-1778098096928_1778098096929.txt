LivingSyria — Syria's definitive online destination combining AI-curated news,

- A responsive bilingual web application (Next.js 14, App Router)

--- AI SERVICE 1: News Ingestion (apps/api/src/services/newsIngestion.ts) ---

This service fetches news from RSS feeds and uses Gemini Flash to generate
bilingual AI summaries. It runs on a schedule and is also manually triggerable.

Dependencies to install in apps/api: rss-parser, node-cron, @types/node-cron

RSS feed sources (hardcode as a constant array to start):
  const RSS_SOURCES = [
    { url: 'https://feeds.bbcarabic.com/bbcarabic/world/rss.xml', nameEn: 'BBC Arabic', nameAr: 'بي بي سي عربي' },
    { url: 'https://www.aljazeera.net/aljazeerarss/a2f35850-5f55-4f9e-83c9-1f202847e1de/d52aebb3-91a3-4dce-9c6f-d1a6b98b1a69', nameEn: 'Al Jazeera Arabic', nameAr: 'الجزيرة' },
    { url: 'https://sana.sy/?feed=rss2', nameEn: 'SANA', nameAr: 'سانا' },
  ];

Service function run():
  1. For each RSS source, fetch and parse feed using rss-parser
  2. For each item in feed:
     a. Skip if article with same sourceUrl already exists in DB
     b. Call Gemini Flash API with this prompt:
        "You are a news editor for a Syrian news platform. Given this Arabic news article,
         generate: 1) a concise Arabic summary (2-3 sentences, summaryAr),
         2) an English translation of the title (titleEn),
         3) an English summary (summaryEn, 2-3 sentences),
         4) an AI-generated Arabic summary for display (aiSummaryAr, 1 sentence),
         5) an English version (aiSummaryEn, 1 sentence),
         6) relevant tags array (aiTags, max 5, in English, from: Politics, Economy,
            Society, Technology, Sports, Culture, Diaspora, Reconstruction).
         Respond ONLY with valid JSON matching this schema:
         { titleEn, summaryAr, summaryEn, aiSummaryAr, aiSummaryEn, aiTags }
         Article title: {item.title}
         Article content: {item.contentSnippet?.slice(0, 500)}"
     c. Parse the JSON response from Gemini
     d. Create NewsArticle record:
        titleAr = item.title
        titleEn, summaryAr, summaryEn, aiSummaryAr, aiSummaryEn, aiTags from Gemini
        coverImageUrl = first image found in item.enclosure?.url or item.content
        sourceUrl = item.link
        sourceName = source.nameEn
        source = NewsSource.RSS_IMPORT
        status = NewsStatus.PUBLISHED
        publishedAt = item.pubDate ? new Date(item.pubDate) : new Date()
        slug = await generateUniqueSlug(item.title) — use same slugify util as listings
     e. Log success or error per article with pino
  3. Log total: articles processed, created, skipped, failed

Error handling: wrap entire run() in try/catch. A single failed article must NOT
stop processing remaining articles. Log errors per article and continue.

Schedule in server.ts (production only):
  import cron from 'node-cron';
  if (config.isProd) {
    cron.schedule('0 * * * *', () => {        // every hour on the hour
      logger.info('Running scheduled news ingestion');
      newsIngestionService.run().catch(err =>
        logger.error({ err }, 'News ingestion cron failed')
      );
    });
    logger.info('News ingestion cron scheduled (hourly)');
  }

Admin manual trigger endpoint (add to admin.ts):
  POST /api/v1/admin/news/ingest  (requireAdmin)
  Calls newsIngestionService.run() and returns { message: 'Ingestion started' }
  Do NOT await — run in background. Return 202 Accepted immediately.

--- AI SERVICE 2: Content Moderation (apps/api/src/services/moderation.ts) ---

Called automatically after every new listing is created (POST /listings).
Uses Gemini Flash to classify listing content as safe or unsafe.

Function moderateListing(listingId: string, titleAr: string, descriptionAr: string):

  1. Call Gemini Flash API:
     "You are a content moderator for a Syrian classifieds platform. Review this
      listing and determine if it is safe and appropriate. Flag content that is:
      spam, illegal goods, offensive, adult content, or political propaganda.
      Listing title (Arabic): {titleAr}
      Listing description (Arabic): {descriptionAr}
      Respond ONLY with valid JSON: { safe: boolean, score: number (0.0-1.0), reason?: string }
      score = confidence that content is safe (1.0 = definitely safe, 0.0 = definitely unsafe)"

  2. Parse JSON response
  3. Update listing in DB:
     moderationScore = result.score
     moderationReason = result.reason ?? null
  4. Auto-approve if safe=true AND score >= 0.85:
     status = ACTIVE
     publishedAt = now()
     Create AuditLog: action='listing.auto_approved', actorId=null, afterState: { moderationScore }
     Create Notification for user: type=LISTING_APPROVED, deepLink=/listings/{slug}
  5. Auto-reject if safe=false AND score < 0.3:
     status = REJECTED
     Create AuditLog: action='listing.auto_rejected', actorId=null, afterState: { moderationScore, moderationReason }
  6. Otherwise (score 0.3-0.85): leave as PENDING_REVIEW for human admin

Call from POST /listings route AFTER creating the listing — fire-and-forget:
  // Non-blocking — do not await, do not fail the request if moderation errors
  moderation.moderateListing(listing.id, listing.titleAr, listing.descriptionAr)
    .catch(err => logger.error({ err, listingId: listing.id }, 'Moderation failed'));

--- AI SERVICE 3: Auto-Translation (apps/api/src/services/translation.ts) ---

Uses Gemini Flash (not GCP Translation API — same key, simpler integration)
to auto-translate Arabic content to English when English fields are missing.

Function translateToEnglish(text: string): Promise<string>:
  Call Gemini Flash:
  "Translate this Arabic text to English. Return ONLY the translated text,
   no explanation, no preamble: {text}"
  Return the response text directly.

Wire into POST /listings (fire-and-forget after listing creation):
  if (!body.titleEn && listing.titleAr) {
    translation.translateToEnglish(listing.titleAr)
      .then(titleEn => db.listing.update({ where: { id: listing.id }, data: { titleEn } }))
      .catch(err => logger.warn({ err }, 'Title translation failed'));
  }
  if (!body.descriptionEn && listing.descriptionAr) {
    translation.translateToEnglish(listing.descriptionAr.slice(0, 1000))
      .then(descriptionEn => db.listing.update({ where: { id: listing.id }, data: { descriptionEn } }))
      .catch(err => logger.warn({ err }, 'Description translation failed'));
  }

Wire into POST /news (admin creates article — fire-and-forget):
  if (!body.titleEn && article.titleAr) { auto-translate titleAr → titleEn }
  if (!body.summaryEn && article.summaryAr) { auto-translate summaryAr → summaryEn }

Admin translate endpoint (for manual use in admin news editor):
  POST /api/v1/admin/translate  (requireAdmin)
  Body: { text: string, targetLang: 'en'|'ar' }
  Returns: { translatedText: string }
  Rate limited: 30 requests/minute per admin user.

--- GEMINI API CALL PATTERN (use for all 3 services) ---

  import { GoogleGenerativeAI } from '@google/generative-ai';
  // Add to apps/api package.json: "@google/generative-ai": "^0.3.0"

  const genAI = new GoogleGenerativeAI(config.googleAi.apiKey);
  const model = genAI.getGenerativeModel({ model: 'gemini-1.5-flash' });

  async function callGemini(prompt: string): Promise<string> {
    const result = await model.generateContent(prompt);
    return result.response.text();
  }

  Always wrap in try/catch. On API error: log with pino, do not crash the request.
  For JSON responses: strip any markdown fences before JSON.parse():
    const clean = text.replace(/```json|```/g, '').trim();
    const parsed = JSON.parse(clean);

SECTION 6 — API SPECIFICATION
================================================================================

Base URL: /api/v1/
All responses: JSON
Auth: Firebase JWT Bearer token in Authorization header
Error format: { error: string, code: string, details?: object }
List responses: { data: [], meta: { total, page, limit, pages, nextCursor: string|null } }

Middleware:
- requireAuth: verifies Firebase ID token, looks up user in DB, attaches req.user.
  Returns 401 if missing/invalid. Returns 403 if status=SUSPENDED.
- optionalAuth: same but continues without error if no token present.
- requireAdmin: use after requireAuth. Returns 403 if role not ADMIN or SUPER_ADMIN.

--- HEALTH ---
GET  /health/live   → { status: 'ok', timestamp }
GET  /health/ready  → checks DB (SELECT 1) + Redis (PING). Returns 503 if either fails.
                      { status: 'ok'|'degraded', checks: { database, redis }, timestamp }

--- AUTH ---
POST   /auth/register  (no auth)
  Body: { idToken, nameAr?, nameEn?, phone?, preferredLang?: 'ar'|'en', locationCountry?: string }
  Verify Firebase idToken server-side. Create user in DB. Return 409 if already exists.
  Response: { user: { id, email, nameAr, nameEn, role, preferredLang, createdAt } }

GET    /auth/me  (requireAuth)
  Response: { user: <full User object> }

DELETE /auth/me  (requireAuth)
  Sets status=DELETED, deletedAt=now(). Response: { message: 'Account deleted' }

PASSWORD RESET (client-side only — no API endpoint needed):
  On the login page, "Forgot password" link opens a modal with an email input.
  Call Firebase client SDK: import { sendPasswordResetEmail } from 'firebase/auth'
  sendPasswordResetEmail(auth, email)
  Show success message: "Password reset email sent" in Arabic and English.
  Firebase handles the email delivery — no backend involvement required.

--- LISTINGS ---
GET  /listings  (optionalAuth)
  Query: page, limit(max 50), category(slug), country, city, minPrice, maxPrice,
         type(FREE|BASIC|FEATURED|PREMIUM), sort(newest|price_asc|price_desc|popular)
  Cache: Redis 30s keyed by serialised query. Invalidate with delPattern('listings:list:*').
  List response: titleAr, price, city, country, primaryImage only. NO descriptionAr/En.
  Response: { data: [], meta: { total, page, limit, pages, nextCursor } }

GET  /listings/:slug  (optionalAuth)
  Cache: Redis 60s. Increment viewCount async (fire-and-forget).
  Full response including: images(all), realEstate, vehicle, job, user(id,nameAr,nameEn,avatarUrl)
  Returns 404 if not found or status=DELETED.

POST /listings  (requireAuth)
  Body: { categoryId(uuid,required), titleAr(required,min3,max200), titleEn?,
          descriptionAr(required,min10), descriptionEn?, price?, currency?(default USD),
          isNegotiable?, isFree?, city?, district?, country?(default SY), lat?, lng? }
  Generate unique slug from titleAr via slugify(). Set status=PENDING_REVIEW.
  Set expiresAt = now + 30 days.
  Invalidate listings list cache on create.
  Response 201: { data: listing }

PATCH /listings/:id  (requireAuth, owner only)
  Validate with UpdateListingSchema — allowlist ONLY these fields:
  titleAr, titleEn, descriptionAr, descriptionEn, price, currency,
  isNegotiable, isFree, city, district, country, lat, lng,
  status(only DRAFT|ACTIVE|SOLD — owner cannot set PENDING_REVIEW or REJECTED)
  NEVER pass req.body directly to Prisma. Always validate first.
  Invalidate listing detail cache + listings list cache on update.

DELETE /listings/:id  (requireAuth, owner or admin)
  Soft delete: status=DELETED, deletedAt=now(). Invalidate caches.

POST /listings/:id/images  (requireAuth, owner only)
  Body: { images: [{ url, thumbUrl?, isPrimary, sortOrder }] }
  Creates ListingImage records. Only one image can have isPrimary=true.

POST /listings/:id/real-estate  (requireAuth, owner only)
  Body: all RealEstateDetail fields. Upsert by listingId.

POST /listings/:id/vehicle  (requireAuth, owner only)
  Body: all VehicleDetail fields. Upsert by listingId.

POST /listings/:id/job  (requireAuth, owner only)
  Body: all JobDetail fields. Upsert by listingId.

--- UPLOAD (SIGNED URL PATTERN — API NEVER PROXIES BYTES) ---

--- CONVERSATIONS ---

  List user's conversations ordered by updatedAt desc.

  Query: cursor(ISO timestamp, optional), limit(default 30, max 50)
  Where: createdAt < cursor if cursor provided.
  Order: createdAt DESC, take limit+1 to determine hasMore.
  Mark received messages as READ. Update member.lastReadAt.
  Response: { data: messages(reversed to ASC), nextCursor: string|null, hasMore: boolean }

  Body: { listingId, message }
  Cannot message own listing (return 400).

    type=NEW_MESSAGE, deepLink=/conversations/{conversationId}
  Response 201: { data: { conversation, message } }

  Body: { body }
  Update conversation.updatedAt.
  Response 201: { data: message }

--- PAYMENTS ---

POST /payments/create-intent  (requireAuth)
  Creates a Stripe PaymentIntent for upgrading a listing.
  Body: { listingId, listingType('BASIC'|'FEATURED'|'PREMIUM') }
  Pricing:
    BASIC   = $500 cents ($5.00)
    FEATURED = $1500 cents ($15.00)
    PREMIUM  = $2500 cents ($25.00)
  Steps:
  1. Verify listing exists and belongs to req.user.id
  2. Create or retrieve Stripe customer for this user (store stripeCustomerId on user)
  3. Create Stripe PaymentIntent with amount, currency='usd', customer id,
     metadata: { listingId, listingType, userId }
  4. Create Transaction record with status=PENDING, stripePaymentId=paymentIntent.id
  Response: { clientSecret: string, transactionId: string }
  The frontend uses clientSecret with Stripe.js to complete payment on the client.

POST /webhooks/stripe  (no auth — verified by Stripe signature)
  IMPORTANT: This route must use express.raw() middleware, NOT express.json().
  Register before the global json() middleware:
    app.use('/webhooks/stripe', express.raw({ type: 'application/json' }), stripeWebhookHandler)

  Verify signature: stripe.webhooks.constructEvent(req.body, sig, STRIPE_WEBHOOK_SECRET)
  Return 400 immediately if signature verification fails.

  Handle these events:
  payment_intent.succeeded:
    1. Find Transaction by stripePaymentId
    2. Set Transaction status=COMPLETED
    3. Set Listing listingType to metadata.listingType
    4. If listingType=FEATURED or PREMIUM: set featuredUntil = now + 30 days
    5. Create Notification for user: type=PAYMENT_RECEIVED,
       titleAr='تم الدفع بنجاح', titleEn='Payment Successful',
       deepLink=/listings/{listing.slug}
    6. Write AuditLog: action='listing.upgraded'

  payment_intent.payment_failed:
    1. Find Transaction by stripePaymentId
    2. Set Transaction status=FAILED

  Always return HTTP 200 to Stripe after handling. Stripe will retry on non-200.

SECTION 7 — app.ts (EXPRESS SETUP — EXACT IMPLEMENTATION)
================================================================================

Build apps/api/src/app.ts with this exact configuration:

1. Middleware order (must be exact):
   helmet() → cors() → compression() → json() → urlencoded() → pinoHttp() → rateLimiter

2. CORS — use origin callback function, NOT static array:
   - Allow: config.webUrl, 'http://localhost:3000', AND null origin (React Native)
   - allowedHeaders: ['Content-Type', 'Authorization', 'X-App-Version', 'X-Platform']
   - exposedHeaders: ['X-RateLimit-Remaining', 'X-Request-Id']

   Implementation:
   const allowedOrigins = [config.webUrl, 'http://localhost:3000'];
   cors({
     origin: (origin, callback) => {
       if (!origin) return callback(null, true); // mobile / no origin
       if (allowedOrigins.includes(origin)) return callback(null, true);
       callback(new Error(`CORS: origin ${origin} not allowed`));
     },
     ...
   })

3. pinoHttp customProps — log X-App-Version and X-Platform on every request:
   customProps: (req) => ({
     appVersion: req.headers['x-app-version'] ?? 'web',
     platform: req.headers['x-platform'] ?? 'web',
   })

4. Rate limiter applied to '/api/' prefix only.

5. IMPORTANT — Stripe webhook must be registered BEFORE the global express.json() middleware,
   using express.raw() instead. Add this line before app.use(express.json(...)):
   app.use('/webhooks/stripe', express.raw({ type: 'application/json' }), stripeWebhookHandler)

6. Register ALL routes:
   app.use('/health', healthRouter)
   app.use('/api/v1/auth', authRouter)
   app.use('/api/v1/listings', listingsRouter)

SECURITY:
- All private endpoints: requireAuth middleware
- Rate limiting: 200/15min general, 20/15min auth endpoints, 50/hr upload
- Helmet.js on all responses
- CORS: origin callback allowing web URL + localhost:3000 + null (mobile)
  allowedHeaders: Content-Type, Authorization, X-App-Version, X-Platform
  exposedHeaders: X-RateLimit-Remaining, X-Request-Id
- ALL inputs validated with Zod. PATCH endpoints: explicit allowlist schemas only.
  NEVER pass req.body directly to Prisma update.
- File uploads: signed URL pattern. API never proxies bytes.
- Prisma ORM only. No raw SQL except SELECT 1 in health check.
- All config from environment variables via requireEnv(). Zero hardcoded values.
- npm audit --audit-level=high in CI pipeline

PERFORMANCE:
- SSR on all public pages (homepage, listings, news)
- Listings list: Redis 30s cache keyed by query. Invalidate on mutation.
- Listing detail: Redis 60s cache. Invalidate on update.
- News articles: Redis 300s cache
- Unread counts: Redis 30s per userId
- List endpoints: strip full body/description — return summaries only
- Messages: cursor-based pagination. All other lists include nextCursor in meta.
- Compression: gzip on all API responses
- Prisma pool: max=5 connections (Cloud SQL f1-micro limit)

RELIABILITY:
- Graceful shutdown: SIGTERM/SIGINT → close server → disconnect DB + Redis → exit
- unhandledRejection: log + process.exit(1)
- Health checks: /health/live (always 200) + /health/ready (checks DB + Redis)
- Redis: Upstash for MVP (free tier). Set REDIS_URL from Upstash dashboard.

INTERNATIONALISATION:
- Locales: ar (default, RTL) and en (LTR)
- URL pattern: /ar/... and /en/...
- next-intl handles routing
- All strings in messages/ar.json and messages/en.json
- Arabic required on all content fields. English optional everywhere.
- Dates: Intl.DateTimeFormat with locale
